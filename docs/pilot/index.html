Empiece aquí

Esta página es el punto de entrada para instalar el sistema de identidad soberana en su propia infraestructura. No necesita una API compartida nuestra: usted genera los secretos y levanta los cuatro componentes.

Modo: SELF-HOST Componentes: 4 Contacto: manuel.rico.tech@gmail.com

Recomendado: instalar con un agente de IA

La forma más rápida y menos propensa a error es preparar AWS (o su máquina) una vez y pedirle a un agente (por ejemplo Cursor) que siga las guías paso a paso. Usted aporta credenciales y secretos; el agente ejecuta la instalación.

  1. Lea esta página (ya está aquí).
  2. Abra el runbook pilot-onboarding-e2e.md — sección Requisitos para instalar con agente en AWS.
  3. Prepare AWS CLI + SSM (o decida instalar Identity en local).
  4. Pase al agente el prompt del runbook y las guías de la carpeta guias/ en orden: Identity → Emisor → Citizen → Verifier.
  5. Haga el smoke test del runbook: solicitar → aprobar → presentar → verificar.

Índice técnico para agentes: AGENTE.md

Los cuatro componentes

El piloto son cuatro repositorios que trabajan juntos. Instálelos en este orden. Cada tarjeta abre la guía Markdown detallada.

1 · Backend

Identity

API NestJS que crea DIDs, recibe solicitudes, firma credenciales, guarda en IPFS y opcionalmente registra en Rootstock. Base del resto.

Puerto 3000 · github.com/Ikabott-MRM/identity Abrir guía Identity
2 · Portal web

IDA Emisor

Back-office Next.js para que un operador revise solicitudes, apruebe o rechace, e inicie la emisión. Login con Cognito propio.

Puerto 3001 · github.com/Ikabott-MRM/IDA-Emisor-Web Abrir guía Emisor
3 · App móvil

Citizen (ciudadano)

Wallet Expo: el ciudadano crea su DID, solicita una credencial, la recibe y la presenta como QR.

Expo · github.com/Ikabott-MRM/SSI-Citizen-App Abrir guía Citizen
4 · App móvil

Verifier (verificador)

Importa la clave pública del emisor, escanea el QR del ciudadano y valida la firma de la credencial.

Expo · github.com/Ikabott-MRM/SSI-Verifier-App Abrir guía Verifier

Si prefiere instalar a mano

También es válido Siga el mismo orden (Identity → Emisor → Citizen → Verifier) usando las guías de guias/. El runbook pilot-onboarding-e2e.md tiene el checklist de secretos (ustedes los generan) y el smoke test final. La instalación con agente sigue siendo la opción recomendada.
